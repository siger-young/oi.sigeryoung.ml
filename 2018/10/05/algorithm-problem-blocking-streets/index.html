Algorithm - Problem - 封锁阳光大学

发表于 2018-10-05 | 分类于 Algorithm ， Problem

封锁阳光大学 洛谷 P1330

问题描述

曹是一只爱刷街的老曹，暑假期间，他每天都欢快地在阳光大学的校园里刷街。河蟹看到欢快的曹，感到不爽。河蟹决定封锁阳光大学，不让曹刷街。

阳光大学的校园是一张由 NNN 个点构成的无向图，NNN 个点之间由 MMM 条道路连接。每只河蟹可以对一个点进行封锁，当某个点被封锁后，与这个点相连的道路就被封锁了，曹就无法在与这些道路上刷街了。非常悲剧的一点是，河蟹是一种不和谐的生物，当两只河蟹封锁了相邻的两个点时，他们会发生冲突。

询问：最少需要多少只河蟹，可以封锁所有道路并且不发生冲突。

输入

第一行：两个整数NNN, MMM

接下来 MMM 行：每行两个整数AAA,BBB，表示点 AAA 到点 BBB 之间有道路相连。

输出

仅一行：如果河蟹无法封锁所有道路，则输出“Impossible”，否则输出一个整数，表示最少需要多少只河蟹。

分析

由题意分析可知，图上的每条边只能被占领一次，而边的占领都源于点的占领，因此一条边所连接的两个点中，只有一个能被占领.
基于这样的一个事实，我们可以先标记某个点为可行，然后将其相邻的点标记为不可行，再将后面那些点的相邻的点标记为可行.
统计所有可行点的个数，即可求解.
根据对称性，将可行和不可行交换，也是所要的解.
因为要求最少的占领数量，最终答案就是可行点和不可行点数目的较小者.
将可行和不可行看成染色过程的两种颜色.

考虑到本题有可能出现非连通图情形，因此采用 DFS 算法来染色，并使用 vis 数组确保唯一访问，将每个连通分量的答案加起来即可.
如果输入数据无解，那么会出现已经访问过的点的颜色和现在要染的颜色不一致的情况.
